8. 网络的吞吐量和网络的时延有何关系
   ```
   网络的吞吐量与时延密切相关，其核心制约关系表现为：
   1.吞吐量增大 -> 排队时延增加
   当网络吞吐量增大时，路由器中等待转发的分组队列变长，导致排队时延显着增加，从而使网络总时延增大。
   2.吞吐量过大 -> 引发网络拥塞
   当吞吐量进一步增加至接近网络饱和时，极易引发网络拥塞，此时整个网络的时延将急剧（大大）增加。
   ```

9. 什么是“无缝的”、“透明的”、“虚拟的”
   ```
   “无缝的”（Seamless）
   核心定义：多个独立的系统或网络联合工作，但对用户而言如同一个整体。
   踩分点：
     网络领域：多网互连使用统一的 IP 协议，屏蔽了网络交界处的“缝隙”，用户视为同一个大网。
     程序领域：多个程序联合完成任务，但对用户只提供一个统一的接口。
   
   “透明的”（Transparent）
   核心定义：某事物物理上实际存在，但其复杂性被隐蔽，对上层或终端用户来说就好像不存在、看不见一样。
   踩分点：如复杂的网络协议层对用户是透明的（用户只需点击发送即可）；或者 PPP 协议的透明传输（不管什么字符都能通过，免去用户对控制字符的顾虑）。
   
   “虚拟的”（Virtual）
   核心定义：某事物在现实物理中并不存在，但通过技术手段，让用户在逻辑上感觉看起来好像存在一样。
   踩分点：典型实例包括虚电路（Virtual Circuit）、虚拟局域网（VLAN）或虚拟机，它们没有对应的独立物理实体。
   ```

10. 为什么协议不能设计成100%可靠
    ```
    1.基本假设前提
      假设存在一个 100% 可靠的协议，双方必须且只需交换 $N$ 次非冗余信息。
    2.最后一次信息的两难困境
      情况 A（不需要确认）：若最后一次（第 $N$ 次）信息不需要对方确认，说明该信息即使丢失也不影响协议的可靠性，即它是冗余的，协议只需 $N-1$ 次即可，与假设矛盾。
      情况 B（需要确认）：若最后一次（第 $N$ 次）信息必须由对方确认才算可靠，则必须引入第 $N+1$ 次确认信息，这与只需 $N$ 次的假设再次矛盾。
    3.两军问题本质与工程结论
      理论本质：对最后一次信息的确认会陷入“确认的确认”的无限递归（死循环），只要信道不可靠，就无法在有限次交换中达成 100% 的绝对同步。
      工程实现：实际协议（如 TCP）无法做到理论上的绝对可靠，只能通过超时重传、三/四次挥手等机制，在工程上无限逼近 100% 可靠。
    ```

12. 试简述分组交换的要点
    ```
    一、 核心机制（4分）
    划分分组，加首部：发送端将较长的报文（Message）划分为若干个等长的数据段。在每个数据段前面加上由控制信息组成的首部（Header）（内含目的地址、源地址、分组编号等关键路由信息），构成一个个分组（Packet）。  
    存储转发（Store and Forward）：作为交换节点（如路由器）不预留物理信道 。当分组到达时，节点先将其接收并存储在缓冲区中，进行差错校验并查找路由表后，再选择一条合适的下一跳链路转发出去 。
    
    二、 核心优势（6分）教材中将分组交换的优点精炼地总结为四个词（每个要点 1.5 分）：
    高效：动态分配传输带宽。分组逐段占用通信链路，不需要事先建立连接并全程独占端到端的物理信道，极大地提高了线路利用率 。
    灵活：独立选择路由。每个分组在传输过程中，可以根据网络当前的拓扑结构、拥塞流量等实时情况，由中间节点独立查找路由表并选择最佳路径。
    迅速：省去了建立连接的排队时延，数据可立即发送；且分组较短，中间节点的存储转发处理速度极快。
    可靠：依靠分布式路由选择机制。当网络中某条链路或某个路由器发生故障时，其他路由器能自动更新路由表并绕道传输，保证网络高生存性。
    
    三、 主要缺点/开销（2分）
    排队时延：分组在各交换节点的缓冲区中排队等待转发，当网络流量突发或拥塞时，会带来一定的排队时延和时延抖动 。
    首部开销：每个分组都必须携带控制信息首部 。频繁发送首部会消耗一定的系统处理能力和链路带宽，带来额外的存储和传输开销。
    ```

13. 试从多个方面比较电路交换、报文交换和分组交换的主要优缺点
    | **比较维度** | **电路交换 (Circuit)**             | **报文交换 (Message)**               | **分组交换 (Packet)**                |
    | ------------ | ---------------------------------- | ------------------------------------ | ------------------------------------ |
    | **连接建立** | **必须**事先建立端到端专用连接     | **无需**建立连接                     | **无需**建立连接                     |
    | **信道占用** | 双方**全程独占**物理线路资源       | 中间节点**存储转发**，动态占用       | 中间节点**存储转发**，动态占用       |
    | **传输单位** | 比特流（连续不间断）               | 完整报文（长度无上限）               | 切分后的**分组**（带首部，长度受限） |
    | **时延特点** | 建立连接慢；但数据传输**时延极小** | 节点整报文存储转发，**时延大且多变** | 分组短小，存储转发**时延较小**       |
    | **利用率**   | **低**（对突发数据极浪费）         | **高**（多路复用）                   | **最高**（统计时分复用）             |

    ```
    二、 主要优缺点总结（背诵核心）
    1. 电路交换
    优点：传输时延极小、无冲突、没有开销首部、保证带宽。
    
    缺点：建立连接时间长、线路利用率低、缺乏灵活性（一路断全路断）。
    
    2. 报文交换
    优点：无需建立连接、线路利用率高、可多目标群发。
    
    缺点：引起较大的排队和存储转发时延、要求交换节点有非常大的缓冲区。
    
    3. 分组交换（目前主流）
    优点：
    高效：动态分配带宽，逐段占用。
    灵活：每个分组独立选择最佳路由。
    迅速：分组短小，可在缓冲区并行存储转发。
    可靠：分布式路由，某段链路断开可自动绕行。
    
    缺点：各节点存在排队时延、每个分组携带首部带来额外开销、可能出现分组丢失或失序。
    ```

14. 互联网基础结构的发展大致分为哪几个阶段？请指出这几个阶段最重要的特点
    ```
    一、 第一阶段：从单个网络向互联网发展（4分）
    核心特点：实现了从单个分组交换网（ARPANET）向互连网络（Internetwork）的演进。
    关键踩点词：1983年 TCP/IP 协议成为 ARPANET 的标准协议，这一年被作为互联网的诞生时间。
    
    二、 第二阶段：建成三级结构的互联网（4分）
    核心特点：围绕美国国家科学基金会 NSFNET 建立了三级计算机网络结构，取代了原本的单级结构。
    三级结构组成：网络分为 主干网、地区网 和 校园网（或企业网）。该阶段后期，互联网开始向私人公司转交经营，迈出商业化第一步。
    
    三、 第三阶段：逐渐形成多层次 ISP 结构的互联网（4分）
    核心特点：政府机构不再负责互联网运营，商用互联网主干网彻底替代 NSFNET，互联网进入全面商业化阶段。
    运行机制：形成了由互联网服务提供商 ISP（Internet Service Provider）多层次覆盖的结构。IP地址按批分给ISP（只批发不零售），用户通过向 ISP 交纳费用接入互联网，现在的互联网由全世界无数大小 ISP 共同拥有。
    ```

15. 简述互联网标准制定的几个阶段
    ```
    一、 互联网草案（Internet Draft）—— 准备阶段（4分）
    核心特征：这是标准制定的初始阶段，其有效期只有六个月。文档性质：在此阶段，该草案还不能算作 RFC 文档。
    
    二、 建议标准（Proposed Standard）—— 形成RFC阶段（4分）
    核心特征：从这个阶段开始，该标准文档正式成为 RFC 文档。流程演变：2011年10月起，为了避免与“互联网草案”混淆，取消了原有的“草案标准”阶段。现在制定流程简化为：建议标准 $\rightarrow$ 互联网标准。
    
    三、 互联网标准（Internet Standard）—— 正式发布阶段（4分）
    核心特征：建议标准经过长期的检验，证明可行后，方可升级为正式的互联网标准。标准编号：正式通过后，会为其分配一个唯一的标准编号，记为 STDxx（一个互联网标准可以关联多个 RFC 文档）。
    ```

17. 计算机网络中的主干网和本地接入网络的主要区别是什么
    ```
    1. 网络角色与位置不同
    主干网：属于计算机网络的核心部分（Core）。
    本地接入网：属于计算机网络的边缘部分（Edge），即连接用户与核心网的“最后一公里”。
    
    2. 组成结构不同
    主干网：由许多高速通信链路和大量高性能路由器组成。
    本地接入网：主要由各种接入传输线路和调制解调设备（如光猫、交换机）组成。
    
    3. 核心功能与主要任务不同
    主干网：负责在网络内部进行长距离、高带宽的分组快速转发与数据传送。
    本地接入网：主要作用仅仅是把终端用户可靠地接入到互联网，满足用户端的数据上传与下载需求。
    ```

18. 互联网的两大组成部分（边缘部分和核心部分）的特点是什么？他们的工作方式各有什么特点
    ```
    一、 边缘部分（Edge）
    组成特点：由所有连接在互联网上的主机（端系统）组成，供用户直接使用。
    主要功能：进行通信（传输数据、音视频）和资源共享。
    工作方式：主要有两种通信方式：
    	C/S 方式（客户-服务器方式）
    	P2P 方式（对等方式）
    
    二、 核心部分（Core）
    组成特点：由大量网络和连接这些网络的路由器组成。
    主要功能：为边缘部分提供服务，即提供连通性和交换。
    工作方式：本质是路由器的工作方式，分为两类任务：
    	转发分组：收到分组后查找路由表，将其转发到下一跳（直接为端系统服务）。
    	交换路由信息：路由器之间不断交换路由信息，以确保路由表与实际网络拓扑一致。
    ```

19. 客户-服务器方式与P2P对等通信方式的主要区别是啥？有没有相同的地方
    相同：

    ```
    底层依赖相同：无论是 C/S 还是 P2P，通信时都需要使用网络核心部分（路由器、交换机等）所提供的基础服务。
    双向通信特性：通信关系一旦建立，双方都可以对等地发送和接收数据。
    本质原理相通：P2P 在本质上仍然基于 C/S 的思想，只是在 P2P 网络中，每一个主机既是客户又是服务器（既向别的节点请求数据，同时也向其他节点提供数据）。
    ```

    区别：

    | **比较维度** | **客户-服务器方式 (C/S)**                                    | **P2P 对等通信方式**                                         |
    | ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
    | **角色定位** | 角色固定。**客户**是服务请求方，**服务器**是服务提供方。     | 角色不固定。节点之间**地位平等**，不显式区分请求方与提供方。 |
    | **工作方式** | 服务器需要**永久在线**并处于被动监听状态；客户需要主动知道服务器的地址。 | 节点动态加入和退出；节点之间直接进行连接和数据交换。         |
    | **硬件要求** | 服务器通常需要**强大的硬件**和高级操作系统支持。             | 对普通主机的硬件和操作系统没有特殊或极高要求。               |

20. 计算机网络有哪些常用的性能指标
    ```
    1. 速率 (Speed)核心定义：连接在网络上的主机在数字信道上传送数据的速率（也称数据率或比特率）。单位：b/s (bps), kb/s, Mb/s, Gb/s。
    2. 带宽 (Bandwidth)核心定义：网络通信线路传送数据的能力，即单位时间内能通过的最高数据率（理想状态下的网络瓶颈值）。单位：b/s (bps)。
    3. 吞吐量 (Throughput)核心定义：单位时间内通过某个网络（或信道、接口）的实际数据量（受限于网络实际路况）。
    4. 时延 (Delay / Latency)核心定义：数据从网络一端传送到另一端所需的时间。
    	核心必背分类：总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延
    5. 时延带宽积 (Delay-Bandwidth Product)核心定义：传播时延与带宽的乘积。
    	公式：时延带宽积 = 传播时延 x 带宽
    	物理意义：代表这条链路能容纳的最大比特数（也称以比特为单位的链路长度，类似水管的体积）。
    6. 往返时间 (RTT)核心定义：从发送方发送数据开始，到发送方收到接收方的确认（ACK）总共经历的时间。
    7. 利用率 (Utilization)
    	核心分类：信道利用率：有数据通过的时间占总时间的百分比（空闲时利用率为0）。
      网络利用率：全网络信道利用率的加权平均值。
    	注意点：利用率不是越高越好，利用率过高会引发严重的排队时延（堵车）。
    ```

    

21. 网络体系架构为什么要采用分层次结构？试着举出一些分层体系结构的思想相似的生活的例子
    ```
    一、 网络体系结构采用分层结构的原因（8分）
    化繁为简（分解复杂性）：将庞大而复杂的网络通信问题，转化为若干较小的、结构单一的局部问题，使其易于研究、设计和处理。（2分）
    各层独立（低耦合与透明性）：高层只需要使用低层提供的接口，而不需要知道低层的具体实现细节。底层对上层是透明的，某一层协议的更新换代不会影响其他层。（2分）
    易于实现与维护：由于每个局部问题规模变小，使得网络硬件和软件的开发、调试、排错以及后期维护变得更加方便和容易。（2分）
    促进标准化与分工协作：明确的层次划分有利于制定统一的国际标准，便于不同厂商、团队之间相互协作与互操作。（2分）
    
    二、 日常生活中的分层体系结构例子（4分）
    以现代快递物流系统为例（如 image_bffbc0.jpg 所示）：
    ```

3. 同步通信和异步通信的区别
   ```
   一、 核心概念与同步机制（4分）
   同步通信：收发双方必须建立统一的时钟（频率相同）。数据以连续的比特流（帧）发送，比特间无间隔。
   
   异步通信：双方无统一时钟。数据以字符或帧为单位发送，间隔任意；靠数据首尾的起始位/停止位（或帧定界符）实现临时同步。
   
   二、 核心技术对比（4分）
   时钟同步：同步通信依靠全网/准同步时钟；异步通信依靠首尾标志临时定界。
   
   传输单位：同步通信为连续数据块（帧）；异步通信为单个字符或独立帧。
   
   传输效率：同步通信辅助开销极小，效率高；异步通信因频繁加标志，额外开销大，效率低。
   
   硬件成本：同步时钟控制复杂，成本高；异步实现简单，成本低。
   
   
   三、 优缺点与应用场景（4分）
   同步通信：效率高、速度快，但技术复杂。适用于高速、大容量、连续的骨干网络传输。
   
   异步通信：简单便宜、容错好，但开销大。适用于低速、突发性强、字符间隔任意的终端通信（如串口、键盘）。
   ```

   

4. 位同步和帧同步的区别
   ```
   一、 核心概念与工作层次（4分）
   位同步（比特同步）：最基本的物理层同步。使接收端时钟与发送端完全一致。
   
   帧同步：属于数据链路层的同步。在接收到的比特流中定位帧定界符，确定帧的准确边界。
   
   二、 核心作用与目的（4分）

   帧同步的目的：确定帧的界限。使接收端能识别出哪些比特构成一个完整的帧，以便进行差错控制（如重传出错帧）或时分复用。
   
   三、 两者的互相关系（4分）
   位同步是基础：接收端必须先实现位同步（知道比特的持续时间），才能正确识别比特流，进而寻找帧定界符实现帧同步。

   ```

```
核心目的：为了及时丢弃有差错的帧，防止接收端收下并上交已经发生传输错误的损坏帧。

主要原因：在传输中出错的帧对接收端主机而言是毫无用处的。如果不进行检测并上交，会白白浪费上层协议和主机的处理资源。

达到效果：虽然不可靠传输协议不负责重传（“只检错，不纠错”），但它可以确保：凡是最终上交给主机的帧，都能以极接近于 1 的概率保证是没有传输差错的。
```



2.除了差错检验，面向字符的数据链路层协议还必须解决哪些特殊问题

```
1.及时丢弃差错帧，保证上交数据的正确性
接收端通过差错检验（如 CRC）可以发现传输中的比特差错，并直接丢弃有差错的帧，确保凡是上交给上层（网络层/主机）的帧都是无比特差错的。

2.避免浪费上层资源与系统开销
若不进行差错检验而直接将错误帧上交，上层协议（如 IP 协议）不仅无法正常解析这些无效数据，还会白白浪费主机的 CPU 处理资源与内存空间。

3.明确分工（“差错检测”是“可靠传输”的基础）

差错检测：负责实现“无比特差错”（只检测并丢弃，不负责重传）。
可靠传输：负责实现“无传输差错”（包含确认和重传机制）。
PPP 和 CSMA/CD 协议将重传工作交给上层（如 TCP）来做，但上层进行重传恢复的前提，是数据链路层能够准确识别并丢弃错误帧。
```

4.是什么原因使以太网有一个最小帧长和最大帧长

```
一、 设置最小帧长的原因（以 64 字节为例）
1.满足 CSMA/CD 协议的冲突检测要求
保证帧的发送时延大于等于争用期（即发送时延 大于等于 2tau），确保站点在帧发送完毕之前能够检测到可能发生的碰撞。
2.区分正常数据帧与碰撞/噪声产生的无效短帧
因发生碰撞而中断的残帧或噪声干扰形成的短帧，其长度均小于最小帧长。接收端只要收到小于最小帧长的帧，即可直接识别并丢弃。

二、 设置最大帧长的原因（以 1518 字节 / MTU 1500 字节为例）
1.保证信道公平竞争与合理时延
防止某个站点发送过长的帧而长时间独占共享信道，导致其他站点产生过长的等待延迟。
2.减少接收缓冲区压力与出错重传开销
过长的帧需要接收端提供极大的缓冲区；且在信道发生误码时，重传长帧会严重浪费网络带宽资源。
```



5.以太网的覆盖范围受限的一个原因是：如果站点之间的距离太大，那么由于信号传输时会衰减得很多因而无法对信号进行可靠的接收。问：如果我们设法提高发送信号的功率，是否可以增大以太网的通信距离

7.数据链路（即逻辑链路）与链路（物理链路）有何区别？“链路接通了”与“数据链路接通了”的区别何在

```
1. 链路与数据链路的区别
链路（物理链路）：指从一个节点到相邻节点的一段物理线路（中间无任何交换节点），仅仅是物理通路的组成部分。
数据链路（逻辑链路）： 在物理链路的基础上，增加了控制数据传输的通信协议以及实现这些协议的软硬件（如网络适配器/网卡）。
数据链路 = 物理链路 + 通信协议（软硬件）

2. “接通”的区别
“链路接通”： 仅指物理线路连通（物理层连通，如网线插好、有电信号），尚不能进行有协议保障的数据传输。
“数据链路接通”： 表示物理线路连通，且数据链路层协议已成功建立/就绪（可进行帧同步、差错控制等可靠的数据帧传输）。
```
